# Ручное тестирование

## Чек-лист для проверки отправки обычного текстового сообщения

1. Проверить, что авторизованный пользователь может открыть существующую комнату.
2. Проверить, что в открытой комнате отображается поле ввода сообщения.
3. Проверить, что поле ввода доступно для ввода авторизованному пользователю.
4. Проверить, что непустое текстовое сообщение можно успешно отправить.
5. Проверить, что отправленное сообщение появляется в ленте комнаты.
6. Проверить, что текст сообщения в ленте совпадает с введённым.
7. Проверить, что после успешной отправки поле ввода очищается.
8. Проверить, что пустое сообщение нельзя отправить.
9. Проверить, что сообщение, состоящее только из пробелов, нельзя отправить.
10. Проверить, что многострочное сообщение успешно отправляется и корректно отображается.
11. Проверить, что сообщение со специальными символами или emoji успешно отправляется и корректно отображается.
12. Проверить, что отправка в несуществующую комнату завершается ошибкой и сообщение не появляется в ленте.
13. Проверить, что отправка без валидного токена завершается ошибкой.
14. Проверить, что повторная отправка с тем же `txnId` не создаёт дубликат сообщения.

---

## Тест-кейсы

### TЕСТ-1 Успешная отправка текстового сообщения в существующую комнату

**Предусловия:**
- существует учётная запись пользователя
- пользователь авторизован
- существует доступная пользователю комната
- поле ввода сообщения активно

**Шаги:**
1. Открыть веб-клиент мессенджера
2. Выполнить логин с валидными данными
3. Открыть существующую комнату
4. Ввести текстовое сообщение, например `Hello QA`
5. Нажать кнопку отправки или выполнить отправку поддерживаемым способом

**Ожидаемый результат:**
- клиент отправляет запрос на бэкенд
- API возвращает успешный ответ
- сообщение появляется в ленте комнаты
- текст сообщения в ленте совпадает с введённым
- поле ввода очищается после успешной отправки
- пользователь не видит ошибку

---

### TЕСТ-2 Попытка отправки сообщения в несуществующую комнату

**Предусловия:**
- у пользователя есть валидный токен авторизации
- для запроса используется `roomId`, который не существует

**Шаги:**
1. Выполнить попытку отправки текстового сообщения в несуществующую комнату
2. Проверить ответ API и поведение клиента

**Ожидаемый результат:**
- API отклоняет запрос с ошибкой, связанной с отсутствием комнаты или невозможностью выполнить отправку
- новое сообщение не создаётся
- сообщение не появляется в ленте
- клиент не показывает ложный успешный результат
- если в UI предусмотрено сообщение об ошибке, оно должно быть понятным

---

### TЕСТ-3 Попытка отправки сообщения без валидной авторизации

**Предусловия:**
- известен `roomId` существующей комнаты
- токен отсутствует, невалиден или истёк

**Шаги:**
1. Выполнить попытку отправки текстового сообщения в существующую комнату без валидной авторизации
2. Проверить ответ API и поведение клиента

**Ожидаемый результат:**
- API отклоняет запрос с ошибкой аутентификации или авторизации
- сообщение не создаётся
- сообщение не появляется в ленте
- клиент либо показывает ошибку авторизации, либо требует повторный логин
- UI не должен показывать отправку как успешную

---

### TЕСТ-4 Повторная отправка с тем же txnId

**Предусловия:**
- пользователь авторизован
- существует доступная комната
- подготовлен валидный body запроса
- используется фиксированное значение `txnId`.

**Шаги:**
1. Отправить текстовое сообщение в существующую комнату с определённым `txnId`
2. Сохранить данные ответа
3. Повторно отправить тот же запрос с тем же `roomId`, `eventType`, body и `txnId`
4. Проверить ответы API и состояние ленты комнаты

**Ожидаемый результат:**
- первый запрос завершается успешно
- повторный запрос с тем же `txnId` не создаёт новое сообщение
- повторный ответ возвращает тот же логический результат, что и первый запрос
- в ленте комнаты отображается только одно сообщение
- дубликат не создаётся
